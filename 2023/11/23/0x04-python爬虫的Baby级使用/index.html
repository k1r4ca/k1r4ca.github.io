点取只能取到第一条，后面即使符合属性要求也是取不到的

基础功能

遍历

一重遍历：

.contents 列表形式输出tag子节点

.parent 输出父节点

.childen 生成器，对tag子节点循环

.string 输出唯一NavigableString子节点内容 或唯一的tag子节点的.string结果

.next_sibling .previous_sibling 输出兄弟节点 （实际很有可能是字符串或空白）

.next_element .previous_element 输出下/上一个解析对象

多重遍历：

.descendants 生成器，对tag所有子孙节点递归

.parents 生成器，对所有父辈节点递归

.next_siblings .previous_siblings 生成器，对所有兄弟节点迭代

.next_elements .previous_elements 生成器，对解析对象迭代（重现解析过程）

过滤器

字符串： 很简单，完全匹配才会输出（不多不少）

如果传字节码默认是UTF-8，可以传Unicode编码来防止解析错误

正则表达式： 其实就是通过match()来匹配

列表：相当于匹配多个字符串，与任一字符串匹配就输出

True：匹配任何值

还有更复杂亿点点的

find_all方法

搜索当前tag所有tag子节点的内容，匹配过滤器条件

name 参数：查找tag，无视字符串节点

keyword 参数： 把该参数作为指定的属性来搜索（看示例会更好懂）

有些属性不能直接搜索，如HTML5的data-*

但可以通过attrs设置字典来搜索

按CSS class搜索：通过class_参数，接受不同过滤器，多值属性可分别搜索

同样可以设置字典进行匹配

string 参数：与name类似，多用于搜索字符串，也可与其他参数混合用于过滤结果

limit 参数：限制返回数，与SQL中limit关键字相似

recursive 参数：默认开启，检索所有子孙节点，设置为false后仅检索子节点

这里就用类似描述一下，具体实现可以实操看看

.insert(*位置*, "...")：插入指定位置

.insert_before("...") insert_after("...")：插入节点前/后

.clear() ：清除tag内容

.extract()：剪切tag

.decompose()：删除tag

.replace_with()：替换tag，原tag返回

.wrap：包装一层tag

.unrap：拆包装，拆掉的tag返回

输出

就简单说一下，其实就是几个不同的输出格式以及输出方法

.prettify()：缩进型树状输出，可以传入编码方式来修改输出

.unicode() .str()：这两个是python方法，后者默认UTF-8但可变

.encode() .decode()：分别获得字节码、Unicode

.get_text()：获取tag中的文本内容，返回Unicode
