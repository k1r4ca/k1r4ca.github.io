本地docker部署试验

先找了某智能写了dockerfile和docker-compose.yml，但是实际上在go中使用sqlite数据库在go mod download后内部会自动生成，并不需要在docker-compose中起一个数据库depend_on设置连接。同时尝试了分步构建container以减小存储占用，未遂，在构建完毕后会自动exit，感觉是构建指令有点问题，没完全起来。所以删改了一下，还是使用了单步构建的方式

前面的分步构建，以后再来探索吧！

浅浅贴一下指令

写好文件就管理员power shell启动，然后docker-compose up -d或者docker-compose up --build一下就好啦！

其实这里还有一点小问题，一会儿说


部署到云服务器

首先是推送到git仓库

先在GitHub上创建一个public仓库，然后复制仓库的url

上一步做完就直接在power shell里执行了，要注意在对应的项目根目录下

先git init

再git remote add origin https://github.com/k1r4ca/MessageBoard.git与仓库建立连接

然后用git add . 将所有文件放入缓存区

这里有个提醒 warning: in the working copy of ‘go.mod’, LF will be replaced by CRLF the next time Git touches it

查了一下是因为不同系统下LF和CRLF两种不同的换行符，一般貌似不太会有影响

这里没有做处理，后面也没出现什么问题（大概

之后是git commit -m "Initial commit"创建commit

最后执行git push -u origin main进行推送，但是果然出了问题

error: src refspec main does not match any
error: failed to push some refs to ‘https://github.com/k1r4ca/MessageBoard.git ‘

查了一下报错是因为分支不对应？

用 git branch -M main处理一下分支，然后再push就成功了

然后terminus，启动！

我是选择在/usr/local下创建了GitHub目录然后用chmod指令将权限改成777，感觉可能不太对，但是就先这样了（

git clone一下然后就是docker-compose up -d了

但是不出所料又出幺蛾子了，一直卡在RUN go mod download了

通过卡在这里联想到应该是docker里面没有设置代理

果然在经历异常漫长的等待后，报错应证了我的想法

RUN go mod download:
390.6 go: github.com/bytedance/sonic@v1.9.1 : Get “https://proxy.golang.org/github.com/bytedance/sonic/@v/v1.9.1.mod “: dial tcp 142.251.42.241:443: i/o timeout

那就把代理改了呗

但是在建构的时候怎么设置代理呢？问了问博学的G老师，G老师跟我说：

救赎之道，就在dockerfile中

大师我悟了！刚刚说的小问题就是这个

ENV GOPROXY=https://goproxy.cn 在FROM下面添加一条这个就好了

重新up up ，建构成功，用sudo docker ps -a查看了一下状态，没什么大问题，端口映射也完成了

但是为什么访问公网IP对应端口、对应路径还是没有连接的状态呢？？？

想到应该是防火墙的问题，查了一下防火墙规则设置的指令

iptables -A INPUT -p tcp --dport 10070 -j ACCEPT 开放了对应端口

很好，还是没用

本来想去看看公网IP还有什么设置的，在腾讯云控制台想起来可以配置防火墙规则

点开一下发现没有设置10070端口开放的规则，自定义添加一下

然后就 愉快地 成功了！看来安全方面做的很好啊，马！
